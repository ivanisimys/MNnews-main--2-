## 📋 Описание проблемы

Пользователь сообщил, что после загрузки сайта на Ubuntu Server не работают:
1. Регистрация новых пользователей
2. Просмотр заявок на регистрацию администратором
3. Принятие/отклонение заявок

## 🔍 Анализ проблемы

### Найденные issues:

1. **Права доступа к файлам (КРИТИЧНО)**
   - Файлы `data/users.json` и `data/news.json` не имели прав на запись для процесса Node.js
   - При переносе с Windows на Linux права доступа не сохраняются корректно

2. **Отсутствие директории data/**
   - Директория `data/` могла отсутствовать при первом запуске на сервере
   - Нет автоматического создания необходимых файлов

3. **Недостаточное логирование**
   - Сервер не выводил подробную информацию об ошибках
   - Сложно было диагностировать проблему

4. **Отсутствие обработки ошибок файловой системы**
   - Нет проверки существования файлов перед чтением/записью
   - Нет проверки прав доступа при запуске

## ⚠️ Важные замечания

1. **Безопасность**: Измените пароль администратора по умолчанию!
2. **Брандмауэр**: Настройте UFW для ограничения доступа к порту 3000
3. **HTTPS**: Для production рекомендуется настроить HTTPS через Nginx + Let's Encrypt
4. **Мониторинг**: Настройте мониторинг доступности сервиса
5. **Обновления**: При обновлении кода делайте бэкап данных

## 🔄 Процесс обновления на сервере

```bash
# 1. Остановить сервис
pm2 stop mnnews
# или
sudo systemctl stop mnnews

# 2. Скопировать новые файлы
scp -r "C:\Users\PC\Documents\MNnews-main (2)\MNnews-main\*" ivan@192.168.1.55:~/mnnews/MNnews-main/

# 3. На сервере перейти в директорию
cd ~/mnnews/MNnews-main/server

# 4. Установить зависимости (если изменились)
npm install

# 5. Запустить сервис
pm2 start mnnews
# или
sudo systemctl start mnnews

# 6. Проверить логи
pm2 logs mnnews
```
