## Что добавлено (оригинальная версия)
- `server/` - backend сервер (`Express`)
- `data/news.json` - данные новостей
- `data/users.json` - локальные пользователи и статусы заявок (`pending/approved/rejected`)
- `js/news-list.js` - загрузка и рендер списка новостей
- `js/news-detail.js` - загрузка и рендер подробной страницы новости
- `pages/News/detail.html?id=<id>` - универсальная detail-страница
- `pages/menu_pages/registration_requests.html` - админ-страница заявок на регистрацию

## Установка и запуск
1. Перейдите в папку сервера:
   `cd "server"`
2. Установите зависимости:
   `npm install`
3. Запустите сервер:
   `npm start`
4. Откройте в браузере:
   `http://localhost:3000`

## Тестовые аккаунты
- Администратор: `admin` / `admin123`
- Игрок: `player1` / `player123`

## Логика регистрации
- Пользователь регистрируется с `username + password + email`
- Аккаунт создаётся со статусом `pending`
- Пока заявка не одобрена, пользователь может войти в аккаунт, но профиль недоступен
- Администратор рассматривает заявки на странице `registration_requests.html`
- Повторные заявки блокируются по `username` и по `email`
- Пароли не хранятся в открытом виде: сервер использует `passwordHash` (`scrypt`)

## API новостей
- `GET /api/health` - проверка сервера
- `GET /api/news` - все новости
- `GET /api/news?category=current|archive1|archive2` - новости по категории
- `GET /api/news/:id` - одна новость по ID

## API авторизации и заявок
- `POST /api/auth/login` - вход
- `POST /api/auth/register` - регистрация (создает `pending` заявку)
- `GET /api/auth/profile/:username` - профиль пользователя
- `GET /api/auth/registration-requests?actor=<admin>` - список pending-заявок
- `POST /api/auth/registration-requests/:username/approve` - одобрить заявку
- `POST /api/auth/registration-requests/:username/reject` - отклонить заявку
- `POST /api/auth/users` - создать пользователя админом (сразу approved)
- `PATCH /api/auth/profile/:username` - редактирование профиля approved-пользователя

## API управления новостями (Админ)
- `POST /api/news` - создать новость
  ```json
  {
    "title": "Заголовок",
    "content": "Текст новости (может быть строкой или массивом)",
    "image": "URL изображения",
    "actor": "username админа"
  }
  ```
- `PUT /api/news/:id` - обновить новость
  ```json
  {
    "title": "Новый заголовок",
    "content": "Новый текст",
    "image": "Новый URL",
    "actor": "username админа"
  }
  ```
- `DELETE /api/news/:id?actor=username` - удалить новость

**Примечание:** Content может быть передан как строка (автоматически разобьется на абзацы по `\n\n`) или как массив строк.

### Основные шаги для Ubuntu:

```bash
# 1. Копирование файлов на сервер
scp -r "C:\Users\PC\Documents\MNnews-main (2)\MNnews-main" ivan@192.168.1.55:~/mnnews/

# 2. Настройка прав доступа (на сервере)
cd ~/mnnews/MNnews-main/server
chmod +x setup_permissions.sh
bash setup_permissions.sh

# 3. Запуск сервера
npm start
```

### Автоматический запуск через PM2:

```bash
# Установка PM2
sudo npm install -g pm2

# Запуск приложения
cd ~/mnnews/MNnews-main/server
pm2 start index.js --name mnnews
pm2 save
pm2 startup systemd
```

### Тестирование API:

```bash
# Запуск тестового скрипта
cd ~/mnnews/MNnews-main/server
chmod +x test_api.sh
bash test_api.sh http://localhost:3000
```

## Быстрый старт для администраторов

1. **Запуск сервера:**
   ```bash
   cd server
   npm start
   ```

2. **Вход как админ:**
   - Через форму входа: `admin` / `admin123`
   - Или через консоль браузера (F12):
     ```javascript
     localStorage.setItem('is_admin', '1');
     localStorage.setItem('name', 'admin');
     location.reload();
     ```

3. **Управление новостями:**
   - Админ-панель: `/pages/menu_pages/admin_panel.html`
   - Или со страницы любой новости (кнопки внизу)

4. **Создание новости:**
   - Нажмите "+ Создать новую новость"
   - Введите текст с абзацами (пустая строка между абзацами)
   - Сохраните

5. **Обновление списка:**
   - На главной странице нажмите "🔄 Обновить список новостей"

## Советы по использованию

💡 **Редактирование текста:** Используйте пустую строку между абзацами для автоматического форматирования

💡 **Быстрое редактирование:** Добавьте `?edit=ID` к URL админ-панели для прямого перехода к редактированию

💡 **Проверка прав:** Убедитесь, что `localStorage.getItem('is_admin') === '1'`

💡 **Отладка:** Откройте консоль браузера (F12) для просмотра ошибок и логов
