# Быстрое исправление проблем с регистрацией и заявками

## Симптомы проблемы
- Регистрация пользователей не работает
- Администратор не видит заявки на регистрацию
- При попытке одобрить заявку возникает ошибка

## Причина
Проблема связана с правами доступа к файлам данных (`data/users.json` и `data/news.json`) при переносе проекта с Windows на Linux.

## Быстрое решение (5 минут)

Подключитесь к серверу и выполните:

```bash
# 1. Переход в директорию проекта
cd ~/mnnews/MNnews-main

# 2. Создание директории data если её нет
mkdir -p data

# 3. Создание файлов данных если их нет
echo '[]' > data/news.json
echo '[]' > data/users.json

# 4. Установка правильных прав доступа
chmod 755 data
chmod 644 data/*.json
chown -R ivan:ivan data

# 5. Переход в директорию сервера
cd server

# 6. Переустановка зависимостей
npm install

# 7. Запуск сервера
npm start
```

## Проверка работоспособности

После запуска сервера откройте другой терминал и проверьте:

```bash
# Проверка health endpoint
curl http://localhost:3000/api/health

# Тест регистрации
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","email":"test@example.com","password":"testpass123"}'
```

Если получили ответ с данными пользователя - регистрация работает!

## Автоматическое решение (скрипт)

Вместо ручного выполнения команд можно запустить готовый скрипт:

```bash
cd ~/mnnews/MNnews-main/server
chmod +x setup_permissions.sh
bash setup_permissions.sh
```

Скрипт автоматически:
- ✓ Создаст необходимые директории и файлы
- ✓ Установит правильные права доступа
- ✓ Проверит установку Node.js и npm
- ✓ Установит зависимости
- ✓ Проверит доступность порта

## Если проблема не решена

### 1. Проверьте логи сервера

```bash
# Смотрите вывод в терминале где запущен сервер
# Или если используете PM2:
pm2 logs mnnews

# Или если используете systemd:
sudo journalctl -u mnnews -f
```

### 2. Проверьте права доступа

```bash
ls -la ~/mnnews/MNnews-main/data/
```

Должно быть:
```
drwxr-xr-x 2 ivan ivan 4096 Apr 15 12:00 .
-rw-r--r-- 1 ivan ivan    3 Apr 15 12:00 news.json
-rw-r--r-- 1 ivan ivan    3 Apr 15 12:00 users.json
```

### 3. Проверьте содержимое файлов

```bash
cat ~/mnnews/MNnews-main/data/users.json
cat ~/mnnews/MNnews-main/data/news.json
```

Оба файла должны содержать корректный JSON (минимум `[]`).

### 4. Проверьте что сервер запущен

```bash
# Проверка процесса
ps aux | grep node

# Проверка порта
netstat -tuln | grep 3000

# Или
lsof -i :3000
```

### 5. Перезапустите сервер

```bash
# Если используете PM2
pm2 restart mnnews

# Если используете systemd
sudo systemctl restart mnnews

# Если запустили вручную
# Нажмите Ctrl+C и запустите снова
npm start
```

## Полное решение с заменой серверного кода

Если простое исправление прав не помогло, замените файл сервера на улучшенную версию:

```bash
cd ~/mnnews/MNnews-main/server

# Резервная копия
cp index.js index.js.backup

# Копирование нового файла (загрузите его на сервер)
cp index_fixed.js index.js

# Перезапуск
npm start
```

Новая версия сервера включает:
- ✓ Улучшенное логирование всех запросов
- ✓ Автоматическое создание директорий и файлов
- ✓ Подробные сообщения об ошибках
- ✓ Проверку прав доступа при запуске
- ✓ Атомарную запись файлов (безопаснее)

## Проверка работы регистрации

1. Откройте браузер: `http://192.168.1.55:3000`
2. Перейдите на страницу регистрации
3. Заполните форму и отправьте
4. Должно появиться сообщение об успешной регистрации

## Проверка работы заявок

1. Войдите как администратор (username: `admin`)
2. Перейдите в "Заявки на регистрацию"
3. Должны отобразиться все ожидающие заявки
4. Нажмите "Принять" или "Отклонить"
5. Статус должен обновиться

## Диагностика конкретных ошибок

### Ошибка: "EACCES: permission denied"

```bash
# Исправление
cd ~/mnnews/MNnews-main
chmod -R 755 data
chown -R ivan:ivan data
```

### Ошибка: "ENOENT: no such file or directory"

```bash
# Создание отсутствующих файлов
cd ~/mnnews/MNnews-main
mkdir -p data
echo '[]' > data/news.json
echo '[]' > data/users.json
```

### Ошибка: "Port 3000 is already in use"

```bash
# Найти процесс
sudo lsof -i :3000

# Остановить процесс
sudo kill -9 <PID>

# Или изменить порт в server/index.js
# const PORT = process.env.PORT || 3001;
```

### Ошибка: "Failed to register" без деталей

Проверьте логи сервера - там должна быть подробная информация об ошибке.

## Полезные команды

```bash
# Просмотр логов в реальном времени
tail -f ~/mnnews/MNnews-main/server/logs.txt

# Проверка размера файлов данных
du -h ~/mnnews/MNnews-main/data/*

# Бэкап данных
cp ~/mnnews/MNnews-main/data/users.json ~/backup/users.json.$(date +%s)

cat ~/mnnews/MNnews-main/data/users.json | python3 -m json.tool
```
