## Как тестировать:

### 1. Запуск сервера
```bash
cd "C:\Users\PC\Documents\MNnews-main (2)\MNnews-main\server"
node index.js
```

### 2. Проверка категорий
- Откройте браузер: http://localhost:3000
- Нажмите на меню "Разделы"
- Должны отображаться категории из `data/categories.json`
- Кликните на любую категорию (кроме "Главная страница")
- Должна открыться страница с новостями этой категории

### 3. Создание новой категории
- Войдите в админ-панель
- Откройте "Управление категориями"
- Создайте новую категорию (например, "Тестовая")
- Проверьте, что она появилась в меню
- Кликните на неё - должна открыться пустая страница категории

### 4. Просмотр новости
- На странице категории кликните на любую новость
- Должна открыться страница с полным текстом новости

### 5. Удаление категории
- В админ-панели откройте "Управление категориями"
- Удалите созданную тестовую категорию
- Проверьте, что она исчезла из меню

## Возможные проблемы:

### Категории не открываются
**Решение:** Убедитесь, что сервер запущен и файлы `category.html` и `category.js` существуют

### Новости не отображаются в категории
**Решение:** Проверьте, что у новостей поле `category` соответствует ID категории

### Ссылки ведут не туда
**Решение:** Проверьте консоль браузера на ошибки, убедитесь что `dynamic-menu.js` загружается корректно

## Структура файлов:

```
MNnews-main/
├── pages/
│   ├── category.html      ← Страница категории
│   └── news.html          ← Страница новости
├── js/
│   ├── dynamic-menu.js    ← Динамическое меню
│   ├── category.js        ← Логика категории
│   └── news-detail.js     ← Логика новости
├── data/
│   └── categories.json    ← Список категорий
└── style.css              ← Стили (обновлены)
```

## API Endpoints:

- `GET /api/categories` - получить список категорий
- `POST /api/categories` - создать категорию
- `DELETE /api/categories/:id` - удалить категорию
- `GET /api/news?category=:id` - получить новости категории
- `GET /api/news/:id` - получить одну новость
